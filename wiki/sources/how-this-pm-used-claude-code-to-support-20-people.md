# How this PM Used Claude Code to Support 20 People

**Channel:** Aakash Gupta | **Published:** 2026-04-07 | [Watch](https://www.youtube.com/watch?v=0UArKLQ6bXA)

## Overview

This video features Hannah Stalberg, a PM at DoorDash and former Google APM, discussing how she uses Claude Code to scale her product management work. She explains her 'Team OS' system, a structured knowledge repository that enables PMs to support larger teams by providing shared context across functions. The video demonstrates how to structure repositories, manage context efficiently, and use advanced planning techniques with AI assistants.

## Key Takeaways

- Create a structured Team OS repository with nested Claude MD files to minimize context usage and improve AI efficiency
- Use doc indexes to help Claude navigate repositories and only access relevant information for each query
- Implement consistent formatting for documents (e.g., customer call summaries) to enable easier AI analysis
- Store plan files in the repository to reuse effective workflows and save time on future similar tasks
- Use plan mode with high effort setting for complex tasks and review agent prompts before execution
- Name and color-code terminal windows to manage multiple concurrent AI sessions effectively

## Notable Insights

- Claude MD files should be very lean - the goal is to minimize context consumption, not maximize it
- Thinking room is the difference between how much information you have and the context window size - crucial for reasoning
- The repository structure is built around the theory of context management: context, context window, compaction, and thinking room
- Everyone on the team should contribute to the repository - it's not just for technical roles
- Plan mode removes Claude's bias for action, allowing for better planning before execution
- Teams become high-performing in the AI-native era when everyone works to improve the repository

## Concepts

- **[[Team OS (Operating System)]]**: A team's knowledge base that stores shared context in one place to help everyone move faster and access information across functional roles
- **[[Context Management]]**: The theory of how LLMs handle information, consisting of context (information in session), context window (capacity), compaction (compression when full), and thinking room (space for reasoning)
- **[[Claude MD Files]]**: Configuration files that guide Claude through repositories, containing doc indexes, team information, and navigation instructions
- **[[Doc Index]]**: A component of Claude MD files that tells Claude how to navigate the repository and where to find different types of information
- **[[Plan Mode]]**: A Claude feature that removes the AI's bias for action, allowing for detailed planning before execution of complex tasks
- **[[Verification]]**: The process of ensuring Claude's work is done correctly, including self-checking mechanisms and source citation
